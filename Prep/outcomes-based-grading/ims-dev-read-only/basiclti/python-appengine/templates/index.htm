Welcome to the IMS BLTI Sample Python Application

This sample application shows how to quickly build a IMS Basic Learning Tools Interoperability application in Python using the Google App Engine environment.

There is an online version of this application at http://lti-gae-python.appspot.com/.

You can use this application as a test end point for LMS/Consumer lauches. If the server is operating in demo mode (empty key store), you can use demo keys of "12345" or "lmsng.school.edu" both with a secret of "secret" at the following endpoints:

  • Best practice no cookie pattern at /launch
  • Using a cookie/sesson pattern with nice URLs at /cookie
  • The Wisdom of Crowds Application at /wiscrowd
Is the server is no longer in demo mode because some keys have been entered into the Key Store, you need to request a key. These keys last until their memory is reused.

You can test launch this application at the IMS-provided Test Basic LTI Consumer. You will need to enter a Launch URL, Key and Secret and Recompute Launch Data before launching back to this application.

You can view the Documentation for this application or documentation on Documentation on Google App Engine.

You can get the source code for this application at http://code.google.com/p/ims-dev/. If you download and install the code, you become the admin and can use all the admin features.
